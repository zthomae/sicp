3.45 Exercise 3.45

Louis Reasoner changes the definition of make-account-and-serializer to automatically serialize deposits and withdrawals. Since serialized-exchange manually serializes the exchange procedure with both serializers, and the exchange procedure will now call serialized functions, a function called during the execution of another function will be mutually exclusive – in other words, the procedure will not be able to terminate. This is an example of deadlock.

Note that we don’t have this problem when nesting two different serializers.

(This exercise anticipates the section on mutexes which directly follows.)
